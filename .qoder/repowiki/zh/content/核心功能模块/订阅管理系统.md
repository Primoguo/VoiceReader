## 简介
本文件围绕"订阅管理系统"展开，聚焦于应用内订阅（StoreKit 2）的加载、检查、购买与恢复流程，以及付费墙展示与功能权限控制。同时梳理订阅状态如何影响 AI 总结、AI 伴读、高品质音色等 Premium 功能的可用性，并给出关键数据流与错误处理策略。**最新更新**：系统已完全切换到生产模式，移除了所有开发模式支持和绕过机制，确保所有用户都必须通过正式的 StoreKit 验证流程才能使用 Premium 功能。

## 核心组件
- 订阅管理器（SubscriptionManager）
  - 职责：加载产品、检查订阅状态、发起购买、恢复购买、维护 isPremium 状态、管理AI功能试用配额
  - 关键点：使用 StoreKit 2；在初始化时异步加载产品与校验状态；提供 @Published 属性供 SwiftUI 响应式更新；**已更新**完全切换到生产模式，移除了testMode支持，新增免费试用次数管理
- 付费墙（PaywallView）
  - 职责：展示 Premium 权益、列出可用订阅产品、触发购买与恢复购买、显示错误信息
  - 关键点：根据 SubscriptionManager 的状态动态渲染；购买成功后自动关闭
- 设置页（SettingsView）
  - 职责：作为 Premium 入口，未订阅用户点击后弹出付费墙；同时承载语音引擎与参数设置
  - 关键点：通过 sheet 展示付费墙；结合 VoiceConfig 切换 TTS 引擎
- 服务器 API 客户端（ServerAPIClient）
  - 职责：封装对中转服务器的 HTTP 请求（总结、伴读、TTS、克隆），统一错误映射
  - 关键点：超时配置、JSON 解析与字段提取、音频数据兼容多种返回格式
- AI 总结服务（AISummaryService）
  - 职责：调用服务端生成摘要，解析结构化结果（正文 + 要点）
  - 关键点：容错处理，当无法解析时回退为整段文本
- AI 伴读服务（CompanionService）
  - 职责：实现交互式对话功能，支持多轮对话上下文、难度自适应和学习状态追踪
  - 关键点：动态上下文注入、会话状态管理、对话历史持久化
- 语音合成服务（SpeechService）
  - 职责：基于 AVSpeechSynthesizer 实现分段朗读、进度回调、跳转、暂停/继续/停止
  - 关键点：按自然断点切分语块；iOS 17+ 优先 Neural TTS；回调驱动 UI 高亮与播放位置更新
- 模型（Document、SummaryResult、VoiceConfig）
  - 职责：持久化文档、存储摘要结果、保存语音配置与引擎选择

## 详细组件分析

### 订阅管理器（SubscriptionManager）
- 设计要点
  - 单例 + MainActor：确保 UI 线程安全与状态一致性
  - 发布属性：isPremium、isLoading、products、currentSubscription 供 SwiftUI 观察
  - 生命周期：初始化即异步加载产品与校验订阅状态
  - **已更新**：完全切换到生产模式，移除了testMode静态属性和相关条件逻辑
  - **新增**：AI功能试用配额管理，使用 UserDefaults 持久化使用次数
- 关键流程
  - 加载产品：调用 StoreKit 获取产品列表，失败则清空并提示
  - 检查订阅：遍历当前授权，若存在有效且未被撤销的交易，标记为已订阅
  - 购买流程：发起购买 → 验证交易 → 完成交易 → 刷新订阅状态
  - 恢复购买：同步 App Store 状态并刷新订阅
  - **新增**：试用次数检查与消耗逻辑
- 复杂度与性能
  - 产品加载与状态检查均为 I/O 操作，采用 async/await 避免阻塞主线程
  - 建议缓存产品列表并在后台定时刷新，减少重复网络请求
  - **新增**：UserDefaults 读写操作轻量高效，不影响用户体验

### AI功能免费试用系统
- 设计要点
  - 为每个AI功能（总结、伴读）独立管理试用次数
  - 使用 UserDefaults 持久化存储，确保应用重启后计数不丢失
  - 仅对非Premium用户生效，Premium用户享有无限使用权
  - 每次成功使用后自动消耗一次试用机会
- 核心逻辑
  - 试用限制：每个功能最多1次免费体验（freeTrialLimit = 1）
  - 权限检查：canUseAISummary 和 canUseAICompanion 属性动态判断
  - 次数消耗：consumeAISummaryTrial() 和 consumeAICompanionTrial() 方法
- 集成方式
  - 在SpeakerViewModel中检查使用权限，决定显示功能还是付费墙
  - 在实际使用时消耗试用次数
  - 与现有订阅系统无缝集成，无需额外UI改动

## 结论
订阅管理系统以 SubscriptionManager 为核心，结合 PaywallView 与 SettingsView 形成完整的付费墙与权限控制闭环。通过 StoreKit 2 的产品加载与交易验证，配合 ServerAPIClient 的后端鉴权与配额策略，实现对 Premium 功能的精细化管控。**最新更新**：系统已完全切换到生产模式，移除了所有开发模式支持和绕过机制，确保所有用户都必须通过正式的 StoreKit 验证流程才能使用 Premium 功能。同时新增了AI功能的免费试用机制，为非Premium用户提供了一次性的体验机会，既保证了用户体验，又为转化提供了合理的引导路径。整体架构清晰、职责分离良好，具备较好的可扩展性与可维护性。
