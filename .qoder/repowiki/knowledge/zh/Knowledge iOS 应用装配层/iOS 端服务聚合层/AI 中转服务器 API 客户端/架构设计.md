单文件实现，通过 `ServerAPIClient.shared` 单例暴露对外接口；内部以 `URLSession` 为唯一网络层，所有 POST 请求经私有 `buildRequest` 构造（固定 Content-Type: application/json），再由三个重载的 `post` 方法按不同返回形态（纯文本 content、DashScope choices 兼容格式、指定字段提取）统一序列化 JSON 并调用 `validateResponse` 做状态码到 `ServerAPIError` 的映射。TTS 路径特殊处理：优先走二进制 audio 流，否则回退到 JSON 中的 `audio_url` 或 base64 字段再二次下载。错误类型集中在同文件末尾的 `ServerAPIError` 枚举中，遵循 `LocalizedError` 协议提供用户可读文案。依赖方向单向：上层业务仅依赖此客户端，不感知底层 URLSession/JSON 细节。
